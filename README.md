
# دفتر الحسابات

تطبيق Flutter لإدارة الحسابات الشخصية والتجارية، مع ميزات مثل تتبع المعاملات، التقارير، النسخ الاحتياطي، وخادم HTTP محلي.

## الميزات

-   **الأمان**: شاشة بصمة الأصابع/كلمة المرور (local_auth, flutter_secure_storage).
-   **قاعدة البيانات**: استخدام Hive (NoSQL) لتخزين البيانات محلياً.
-   **إدارة العملاء**: عرض، إضافة، تعديل، وحذف العملاء مع تفاصيل المعاملات.
-   **التقارير**: تقارير متنوعة (إجمالية، تفصيلية، شهرية، تصنيفات) مع إمكانية التصدير إلى PDF, Excel, CSV.
-   **النسخ الاحتياطي والاستعادة**: يدوي وتلقائي لبيانات التطبيق.
-   **استيراد البيانات**: من ملفات CSV أو JSON.
-   **خادم HTTP محلي**: لاستعراض البيانات من الكمبيوتر عبر المتصفح.
-   **الإعدادات**: تخصيص شامل للتطبيق (بيانات الشركة، العملات، التصنيفات، الإشعارات، إلخ).
-   **الإشعارات المحلية**: تذكيرات بالأقساط المتكررة.
-   **تحديث التطبيق**: التحقق من وجود تحديثات عبر GitHub Releases.
-   **مشاركة ودعم**: مشاركة التطبيق والتواصل مع الدعم عبر واتساب.

## متطلبات التشغيل

-   [Flutter SDK](https://flutter.dev/docs/get-started/install) (الإصدار 3.0.0 أو أحدث).
-   [Android Studio](https://developer.android.com/studio) أو [VS Code](https://code.visualstudio.com/) مع إضافات Flutter.

## كيفية إعداد وتشغيل المشروع

1.  **استنساخ المستودع (Clone the repository)**:

    ```bash
    git clone [رابط مستودع GitHub الخاص بك]
    cd دفتر_الحسابات
    ```

2.  **الحصول على التبعيات (Get dependencies)**:

    ```bash
    flutter pub get
    ```

3.  **توليد ملفات Hive Adapters**:

    يستخدم هذا المشروع Hive لتخزين البيانات، ويتطلب توليد ملفات `*.g.dart` تلقائياً. قم بتشغيل الأمر التالي:

    ```bash
    flutter pub run build_runner build --delete-conflicting-outputs
    ```

    *ملاحظة*: يجب تشغيل هذا الأمر في كل مرة تقوم فيها بتعديل ملفات `account.dart`, `transaction.dart`, أو `settings.dart`.

4.  **تشغيل التطبيق (Run the app)**:

    ```bash
    flutter run
    ```

    أو قم بتشغيله من خلال بيئة التطوير المتكاملة (IDE) الخاصة بك (Android Studio/VS Code).

## بناء ملف APK (لـ Android)

لإنشاء ملف APK جاهز للتثبيت، استخدم الأمر التالي:

```bash
flutter build apk --release
```

سيتم العثور على ملف APK في المسار `build/app/outputs/flutter-apk/app-release.apk`.

## GitHub Actions (للبناء التلقائي)

تم تضمين ملف GitHub Actions (`.github/workflows/build.yml`) يقوم تلقائياً ببناء ملف APK عند كل عملية دفع (push) إلى الفرع الرئيسي (main)، ويمكنه أيضاً إنشاء إصدار (Release) على GitHub مع ملف APK المرفق عند إنشاء وسم (tag).

**للاستفادة من ميزة التحديث التلقائي:**

1.  قم بتغيير `[YOUR_GITHUB_USERNAME]` و `[YOUR_GITHUB_REPO]` في ملف `lib/utils/constants.dart` إلى اسم المستخدم واسم المستودع الخاص بك على GitHub.
2.  عند إنشاء إصدار جديد (Release) على GitHub، تأكد من أن اسم الإصدار (tag) يتبع تنسيقاً مثل `v1.0.0`.
